# 目的地検索アプリ
目的地提案アプリ 構想まとめ

## 1. コンセプト

**「迷う時間をゼロにして、外出する場所を決める/家を出るきっかけを作る」**
「行きたい場所はある/外出したい気持ちはあるけれど、どこに行こうか迷っているうちに時間が過ぎてしまう」という心理的ハードルを、【保存済みの場所 × 空き時間 × ランダム選出】の掛け合わせで解決する。

### ユーザー

- 外出したい気分だけどどこに行こうか迷って出かける機会を逃す人
- デート先を迷うカップル

## 2. 既存アプリ（Googleマップ等）との差別化

目的地が決まっている人を「誘導」するのではなく、**「意思決定を代行・加速」**することに特化

- **場所のストック:** 場所と選択式でどのくらい滞在したいかの時間を登録
- **行く場所の提示:** 「移動往復 ＋ (滞在時間) ≦ 自由時間」という独自の計算ロジックによって行く場所のフィルタリング。
- **選択肢の制限:** あえて候補をたくさん並べず、ランダムに「1つ」を提示することで、比較による迷いを物理的に排除する。ユーザー自身に選ばせない。

## 3. MVP（最小限の機能構成）

開発初期に実装すべきコア機能。

| 機能 | 詳細 |
| --- | --- |
| **場所登録** | スポット名もしくは位置情報、滞在したい時間を保存。
滞在時間は以下の選択式
**「サクッと」**（滞在目安：〜30分程度）**「ゆっくり」**（滞在目安：1〜2時間程度）**「たっぷり」**（滞在目安：3時間〜） |
| **時間入力検索** | 移動時間を入力すると、現在地からの移動時間内に到着できる場所を探す |
| **場所抽出(ランダム)** | 検索された条件に合う候補から1つだけをランダムで提示。「行く」か「再抽選」かを選ぶだけのシンプルな操作。検索結果がない場合は1番条件に合う場所を表示 |
| **行った登録(MVPではないかも)** | 行った場所を登録する
被ることを防ぐ
検索の設定で行ったことあるところも表示するかは選択可能にする |

## 4. ユーザー体験（UX）のフロー

1. **保存:** 普段から気になる場所（カフェ、練習場、ショップなど）を登録
2. **入力:** ふと時間が空いた時、アプリに「今から3時間で行ける場所」と入力
3. **提案:** アプリが「ここなら3時間で行けます」と、条件に合う候補の中から1ヶ所をランダムに表示
4. **出発:** 比較検討のストレスなく、「よし、行こう」と家を出る

## 5. 今後の拡張案（Ver. 2.0〜）

- **気分タグ:** 「無心になりたい」「体を動かしたい」など、その時のマインドに合わせた絞り込み。
- **有効期限・シーズン設定:** 期間限定のイベントや、季節限定（「春に行きたい」など）の場所を優先的に提案。
- **ゲーミフィケーション:** 外出を達成するごとに記録が残り、自分の「行動範囲」や「充実度」が可視化される仕組み。
