差分確認からPR作成までを行います。

## 手順

### 1. 差分確認
```
git status
git diff
```
変更内容を確認し、ユーザーに見せてください。

### 2. ステージング
変更ファイルを確認した上で `git add` でステージングしてください。
機密ファイル（.env など）は含めないよう注意すること。

### 3. コミット
差分の内容をもとに適切なコミットメッセージを作成し、コミットしてください。
コミットメッセージは日本語で作成してください。

### 4. ブランチ確認・作成
現在のブランチを確認してください。

- **フィーチャーブランチにいる場合**: そのままプッシュします。
- **`main` ブランチにいる場合**: 以下の2択をユーザーに選ばせてください。
  1. `main` に直接プッシュしてPRは作成しない
  2. フィーチャーブランチを作成してPRを作成する

選択肢1の場合は `git push origin main` してPR作成をスキップし、完了を伝えてください。
選択肢2の場合はフィーチャーブランチを作成してからプッシュしてください。

```
git push -u origin <branch-name>
```

### 5. PR作成
`gh pr create` を使って PR を作成してください。

- タイトル: コミット内容を要約した簡潔なもの（70文字以内）
- 本文: 変更内容のサマリーとテスト観点をMarkdownで記述

```
gh pr create --title "<タイトル>" --body "<本文>"
```

PR の URL を最後にユーザーに伝えてください。
